# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it privately.

**Do not file a public GitHub issue for security problems.**

Please send a detailed report to the maintainer via GitHub: [@KooshaPari](https://github.com/KooshaPari).

Include in your report:

- A description of the vulnerability and its impact
- Steps to reproduce or a proof-of-concept
- The affected version(s) or commit SHA(s)
- Any known mitigations or workarounds

You can expect an initial acknowledgement within 72 hours. We will work with you
to understand the issue, coordinate disclosure, and release a fix as appropriate.

## Supported Versions

Only the latest release on the `main` branch receives security fixes. Please
upgrade to the most recent version before reporting an issue.
